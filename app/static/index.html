Retrieval Strategy

                            
Three-pronged hybrid retrieval merges results from multiple strategies:

                            
    
                                
  • Name matching — Regex detects LAPACK routine names (e.g., DGESV) in the query and fetches exact matches via Qdrant payload filter.
    • 
                                
  • LLM query expansion — For conceptual queries ("How does SVD work?"), the LLM identifies 5-8 relevant routine names, each searched individually.
    • 
                                
  • Call-graph following — One-hop expansion: routines called by name-matched results are also retrieved, surfacing implementation dependencies.
    • 
                                
  • Vector similarity — Standard cosine similarity search fills remaining top-k slots, ensuring broad coverage.
    • 
                            

                        


                        

                            
Components

                            
    
                                
  • FastAPI — Web server serving the UI and REST API
    • 
                                
  • fparser — AST-based Fortran parsing (fixed-form .f + free-form .f90) with RAW fallback
    • 
                                
  • OpenAI — text-embedding-3-small (1536-dim) for embeddings, gpt-4o-mini for generation
    • 
                                
  • Qdrant Cloud — Managed vector database with payload indexes for name filtering
    • 
                                
  • tiktoken — Token counting for chunk splitting and context budget enforcement
    •

  • Railway — Auto-deploys from GitHub on push. Single Dockerfile-based service.
